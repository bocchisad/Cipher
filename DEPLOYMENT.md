# 🚀 Развертывание Cipher P2P Messenger

## Проблема с WSS подключением

Ошибка `Firefox не может установить соединение с сервером wss://...` означает, что WebSocket сервер на Replit не отвечает.

## Решение: Что мы изменили

### 1. **Исправлены порты сервера** (server.js)
- Изменено слушание на порт `5000` вместо `443`
- Replit автоматически обрабатывает HTTPS/WSS на своей стороне
- Сервер слушает на `0.0.0.0` для всех интерфейсов

### 2. **Улучшена логика подключения клиента** (index.html)
- **10 попыток переподключения** (было 5)
- **Exponential backoff** - задержка между попытками растет
- **Лучшее логирование** - видно что происходит в консоли
- **Автономный режим** - работает без сервера

### 3. **Добавлен artico fallback**
- Если WebSocket не работает после 10 попыток
- Клиент пробует инициализировать artico P2P
- Используется для автономной синхронизации

### 4. **Улучшена обработка сообщений**
- Сервер отправляет `server-ready` при подключении
- Лучше обработка `auth-ok` сообщений
- Правильное логирование всех событий

## 📋 Чек-лист развертывания

### На локальной машине:
```bash
# 1. Установить зависимости
npm install

# 2. Запустить сервер
npm start
# или: node server.js
```

Сервер должен запуститься на `ws://localhost:5000`

### На Replit:
1. Загрузить мой проект на Replit
2. Запустить `npm install` в консоли Replit
3. Нажать зеленую кнопку "Run"
4. Скопировать URL вашего Replit приложения
5. **ВАЖНО**: Обновить `serverAddress` в `index.html` строка 425:
   ```javascript
   let serverAddress = 'wss://YOUR-REPLIT-URL.repl.co';
   // Например: 'wss://cipher-p2p-messenger.username.repl.co'
   ```

## ⚡ Быстрое исправление

Если все еще не работает:

1. Убедитесь что `npm install` выполнен и установлены все пакеты
2. Проверьте URL в `index.html` - он должен быть точно как в Replit
3. Перезагрузите страницу браузера (Ctrl+F5)
4. Очистите localStorage: откройте dev tools → Application → localStorage → удалите `cipherAuth`
5. Попытайтесь зарегистрироваться заново

## 📞 Поддержка

- If WSS still fails after 10 retries → автономный режим (работает локально)
- все сообщения сохраняются в IndexedDB (браузер)
- Когда сервер вернется → сообщения синхронизируются автоматически
